Deduplication

LumiVault prevents wasted storage by detecting exact duplicates and near-duplicates before archiving.

Exact Deduplication (SHA-256)

Every imported file is hashed with SHA-256. If a file with the same hash already exists in the catalog, it is skipped. This catches byte-for-byte identical files regardless of filename.

Near-Duplicate Detection

Enable Near-Duplicates during export to detect visually similar images that aren't byte-identical (e.g., slightly different crops, edits, or re-exports).

How It Works

  1. LumiVault computes a perceptual hash (pHash) for each image — a compact fingerprint of the visual content.
  2. Perceptual hashes are compared using Hamming distance (number of differing bits).
  3. Pairs with a Hamming distance below the threshold are flagged as near-duplicates.

Reviewing Near-Duplicates

After export, the completion screen shows near-duplicate pairs with:

  • Side-by-side thumbnail previews.
  • Hamming distance score (lower = more similar).

You can also review near-duplicates later via the Near-Duplicates toolbar button.

Note: Near-duplicate detection is informational. LumiVault flags potential duplicates but does not automatically skip them. You decide which to keep.
