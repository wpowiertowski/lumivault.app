Integrity & Reconciliation

LumiVault provides tools to verify that your archived photos remain intact and that all storage locations are in sync.

Full Catalog Scan

Open Settings > Integrity to scan your entire archive:

  1. LumiVault checks each file's declared storage location on connected volumes.
  2. Detects missing files, orphaned files, and B2 discrepancies.
  3. With Verify file hashes enabled, computes a fresh SHA-256 hash for every file and compares it against the catalog to detect corruption.

Reconciliation

The scan detects several types of discrepancies:

  • Missing files — files listed in the catalog but not found on a volume.
  • Extra files — files on a volume that aren't in the catalog.
  • B2 discrepancies — files missing from or extra in B2 cloud storage.
  • Hash mismatches — files that exist but have different content than expected (corruption).

Auto-Repair

Enable Auto-repair in the Integrity tab to automatically fix corrupted files during a scan. LumiVault uses a two-step repair strategy:

  1. Copy from healthy volume — if the same file exists uncorrupted on another connected volume, the healthy copy replaces the corrupted one.
  2. PAR2 recovery — if no healthy copy is available, LumiVault uses PAR2 parity data to reconstruct the corrupted blocks via Reed-Solomon error correction.

Repair results are shown inline — green checkmark for repaired files, red X for unrecoverable ones.

Per-Album & Per-Image Verification

You can verify and repair individual albums or images without running a full catalog scan:

  • Album — right-click an album in the sidebar and choose Verify & Repair.
  • Image — right-click a photo in the grid and choose Verify & Repair.

Both open a sheet that automatically verifies file hashes on all connected volumes and repairs any corruption found, using the same copy-from-volume / PAR2 strategy as the full scan.

Tip: Run integrity verification periodically (e.g., monthly) to catch corruption early, while PAR2 repair is still possible.
