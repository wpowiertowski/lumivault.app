Backblaze B2 Cloud Storage

LumiVault can upload archived photos to Backblaze B2 for off-site cloud backup. Uploads include SHA-1 verification to ensure data integrity.

Setting Up B2

  1. Create a Backblaze B2 account if you don't have one.
  2. Create a bucket in the B2 dashboard. Note the Bucket ID and Bucket Name.
  3. Create an application key with read/write access to your bucket. Note the Key ID and Application Key.
  4. In LumiVault, open Settings > B2 (⌘,).
  5. Enable B2, enter your credentials, and click Save.
  6. Click Test Connection to verify your credentials work.

Uploading

During export, enable the B2 Upload toggle. After files are copied to local volumes, LumiVault uploads each file to B2 with:

  • SHA-1 verification — B2 verifies the uploaded file matches the SHA-1 hash sent with the request.
  • Same folder structure — files are stored in B2 using the same date/album path as local volumes.
  • PAR2 files included — if PAR2 is enabled, parity files are uploaded alongside the originals.

Catalog Backup

LumiVault automatically uploads catalog.json to B2 after each export, providing an off-site catalog backup.

Deletion

When you delete albums or photos, LumiVault removes the corresponding files from B2 as well as from local volumes.

Important: B2 storage incurs costs based on storage used and downloads. Review Backblaze pricing before enabling large uploads.
