PAR2 Error Correction

PAR2 (Parity Archive Volume 2) uses Reed-Solomon error correction codes to protect your archived photos against bit rot and storage corruption.

How It Works

  1. During export, LumiVault generates PAR2 parity files alongside each image.
  2. Parity files contain redundant data computed from the original file using Reed-Solomon coding.
  3. If a file is later damaged (bit rot, bad sectors), the parity data can reconstruct the missing or corrupted bytes.

Enabling PAR2

Toggle PAR2 in the export configuration step, or set it as default in Settings > Export.

Redundancy Percentage

Configure the redundancy percentage in Settings > General. This controls how much parity data is generated:

  • 5% — recovers from minor corruption (few bad bytes).
  • 10% (default) — good balance of protection and file size overhead.
  • 20%+ — maximum protection, larger parity files.

PAR2 Files

For each archived image, PAR2 generates:

  • photo.jpg.par2 — index file with verification checksums.
  • photo.jpg.vol00+01.par2 — recovery volume(s) with parity data.

These files are stored alongside the originals on external volumes and uploaded to B2 if enabled.

GPU Acceleration

LumiVault uses Metal GPU acceleration for PAR2 generation when available, significantly speeding up parity computation on Apple Silicon Macs.

Tip: PAR2 protects against silent data corruption (bit rot) that can occur on any storage medium over time. It's especially valuable for long-term archival storage.
